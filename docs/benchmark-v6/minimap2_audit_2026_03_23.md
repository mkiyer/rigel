# Minimap2 Short-Read RNA-Seq Alignment — Audit & Benchmark Results

Date: 2026-03-23  
Benchmark: pristine (no gDNA, no nRNA, 10M fragments, ss=0.95, basic transcript filter)

---

## 1. Minimap2 Configuration Audit

### Issues Found and Fixed

| Issue | Old (broken) | Fixed | Impact |
|-------|-------------|-------|--------|
| Secondary alignments | `-N 10` | `-N 20` | **CRITICAL** — was the dominant source of error |
| Junction flag | `-a -x splice:sr --junc-bed FILE` | `-ax splice:sr -j FILE` | Minor (both flags appear equivalent) |
| Error logging | minimap2 stderr silently swallowed | Logged at DEBUG level | Operational |
| BED fallback | `build_minimap2_bed` crashed if k8 unavailable | Falls back to annotation.bed12 | Robustness |

### Correct minimap2 Command (per Heng Li blog, 2025-04-18)

```bash
# 1-pass with annotation (recommended)
minimap2 -ax splice:sr \
    -j annotation.bed \        # BED12 splice junctions (paftools.js gff2bed or rigel write_bed12)
    --secondary=yes -N 20 \    # Report up to 20 secondary alignments per read
    -t 8 \
    genome.fa r1.fq.gz r2.fq.gz | \
  samtools sort -n -o output.bam

# Index build (one-time, reusable)
minimap2 -x splice:sr -d genome.mmi genome.fa
```

Key notes:
- `splice:sr` is the short-read RNA-seq preset (minimap2 ≥ v2.29). Do NOT use `splice` (long-read preset).
- `-j annotation.bed`: BED12 junction annotation boosts alignment accuracy at annotated splice sites,
  critical for reads spanning tiny exons (< 60bp). Generated by `paftools.js gff2bed` (preferred,
  includes all isoforms) or rigel's `write_bed12` (fallback, basic filter).
- `-N 20`: 10 was insufficient for highly-duplicated genes (GAPDH with 64 pseudogenes, EEF1A1 with 74).
- 2-pass alignment (`--write-junc` / `--pass1`) is NOT available in v2.30; requires ≥ v2.31.
- The junction annotation file for the benchmark is at `benchmark_pristine/minimap2_annotation.bed`
  (generated by `paftools.js gff2bed` from the full GTF, 457k+ transcripts).

## 4. Top 5 Error Genes (Post-Fix)

### Rank 1: ENSG00000280800 / ENSG00000281181 / ENSG00000280614 — Gene Triplication Locus

| Transcript | Truth | Oracle | MM2 | MM2/Truth |
|-----------|-------|--------|-----|-----------|
| ENST00000631211.1 (ENSG00000280800) | 21,430 | 21,430 | 7,047 | 0.329 |
| ENST00000627981.1 (ENSG00000281181) | 0 | 0 | 7,250 | spurious |
| ENST00000625598.1 (ENSG00000280614) | 0 | 0 | 7,167 | spurious |

**Root cause**: Three near-identical gene copies. The simulation generates reads from ENSG00000280800,
but minimap2 correctly maps them to all three loci with equal or near-equal alignment scores.
Since NM≈0 at all three loci, rigel's edit-distance penalty cannot differentiate them. The oracle
correctly assigns all reads to ENSG00000280800. This is a **fundamental identifiability failure**
for duplicated genes with high sequence identity.

**Fix options**: None at the alignment level. Would require read-level haplotype phasing or
excluding duplicated loci from quantification.

---

### Rank 2: AP2M1 (ENSG00000161203.14) — Within-Gene Isoform Confusion

| Transcript | Truth | Oracle | MM2 | MM2/Truth |
|-----------|-------|--------|-----|-----------|
| ENST00000292807.9 | 2,352 | 2,354 | 190 | 0.081 |
| ENST00000382456.7 | 108 | 167 | 1,932 | **17.9×** |
| ENST00000621863.5 | 202 | 217 | 400 | 1.98× |

**Root cause**: Within-gene isoform confusion. The dominant isoform (ENST00000292807.9, truth=2352)
is catastrophically under-counted while a minor isoform (ENST00000382456.7, truth=108) is
over-counted 18×. The oracle correctly distinguishes these. This is an alignment artifact where
minimap2 consistently places dominant-isoform reads into the wrong isoform, possibly due to
junction ambiguity at a specific exon boundary. The specific splicing difference between these
isoforms creates a fragile alignment anchor that minimap2 can't correctly resolve.

**Investigation needed**: Examine the junction structure of ENST00000292807.9 vs ENST00000382456.7.
Likely has a tiny exon or a junction near a repeat element.

---

### Rank 3: SLC25A6 (ENSG00000169100/ENSG00000292334) — Pseudoautosomal Region (PAR) Duplication

| Transcript | Truth | Oracle | MM2 | MM2/Truth |
|-----------|-------|--------|-----|-----------|
| ENST00000381401.11 (chrX copy) | 2,328 | 2,328 | 1,148 | 0.493 |
| ENST00000711214.1 (chrY copy) | 0 | 0 | 1,180 | spurious |

**Root cause**: SLC25A6 lies in the pseudoautosomal region (PAR1), where chrX and chrY are
essentially identical. Simulated reads from the chrX copy are split ~50/50 between chrX and chrY
by minimap2. Oracle is aware of the original fragment position and ignores the chrY copy.
This is a **fundamental ambiguity for ALL PAR genes**.

**Fix options**: Mask or exclude PAR regions from quantification, or treat PAR duplicates as a
single locus (merge chrX/chrY PAR gene pairs).

---

### Rank 4: POLR2A (ENSG00000181222.19) — Single-Transcript Alignment Failure

| Transcript | Truth | Oracle | MM2 | MM2/Truth |
|-----------|-------|--------|-----|-----------|
| ENST00000674977.2 | 2,261 | 2,261 | 769 | 0.340 |

**Root cause**: A single-transcript gene where oracle is perfect but minimap2 loses 67% of fragments.
This suggests POLR2A has one or more micro-exons (similar to GAPDH's 52bp problematic exon) where
reads fail to anchor correctly with minimap2, even with `splice:sr -j annotation`. POLR2A has
~33 exons and is known for complex splicing. The annotation BED does contain the POLR2A junctions,
but the anchor may still be too short for some junctions.

**Investigation needed**: Map transcript structure, identify which junctions are causing failures.

---

### Rank 5: RPL21 (ENSG00000122026.11) — Ribosomal Protein Pseudogene Absorption

| Transcript | Truth | Oracle | MM2 | MM2/Truth |
|-----------|-------|--------|-----|-----------|
| ENST00000311549.11 | 8,188 | 8,213 | 5,640 | 0.689 |
| RPL21P16 (ENST00000495531.1) | 0 | 0 | 2,444 | spurious |

**Root cause**: Ribosomal protein pseudogene absorption. RPL21P16 is a processed pseudogene of
RPL21. Simulated reads from the true RPL21 transcript multimap to RPL21P16. With `-N 20`, rigel
can now see both RPL21 and RPL21P16 as candidates. Rigel's NM penalty should favor RPL21 (0 NM)
over RPL21P16 (≥1 NM), but RPL21P16 is a processed pseudogene with high sequence similarity
so NM differences may be small (1-3 per 150bp read).

**Fix options**: Increase `mismatch_log_penalty` weight; or better MD-tag parsing that accounts
for paired-end overlap double-counting (currently unimplemented).

## 5. Error Category Summary

| Category | Examples | Root Cause | Fixable? |
|----------|---------|-----------|---------|
| Gene triplication/duplication | ENSG00000280800 cluster | Perfect sequence identity across copies | No — fundamental |
| PAR genes | SLC25A6 | chrX=chrY in PAR | Partial — merge PAR pairs |  
| Within-gene isoform confusion | AP2M1, SEPTIN7, UBB | Junction misalignment → wrong isoform | Partial — better annotation |
| Single-gene alignment failure | POLR2A, OAZ1, RTRAF | Micro-exon anchor failure | Partial — `splice:sr -j` helps |
| Ribosomal/ubiquitin pseudogenes | RPL21, UBB | Pseudogene absorbs reads (low NM diff) | Partial — stronger NM penalty |

## 6. Next Steps

### High Priority
1. **AP2M1 and SEPTIN7 isoform confusion**: Investigate the junction structure. These are pure
   minimap2 alignment errors (oracle is correct). Examine whether 2-pass alignment (v2.31+)
   would fix them. Consider investigating `--junc-bonus` parameter.

2. **POLR2A alignment failure**: Identify the problematic exons. Check if the annotation BED
   contains ALL POLR2A junctions correctly. Single 150bp reads may still fail to anchor at
   exons < 30bp.

3. **PAR gene handling**: Consider a dedicated PAR handling strategy — either mask PAR in
   the reference (use hrGRCh38 PAR-masked reference), or merge chrX/chrY PAR gene pairs
   in the rigel index.

### Low Priority  
4. **Stronger NM penalty**: For pseudogene multimappers with 1-5 NM differences, the current
   `mismatch_log_penalty = log(0.1) = -2.3` per mismatch may not be sufficient. The residual
   9.5% error in the 1,001-10k abundance bucket (1.39× vs oracle) is largely from this category.

5. **2-pass alignment**: Once minimap2 ≥ v2.31 is available, implement 2-pass alignment
   (`--write-junc` + `--pass1`) which should further improve junction recovery.
